Layer0 Setup (l0-setup) command-line interface reference#

The l0-setup application is designed to be used with one of several commands: apply, backup, destroy, endpoint, plan, restore, terraform, and vpc. These commands are detailed in the sections below.

General Usage#

l0-setup [--version] command [options] [parameters]

Apply#

The apply command is used to create and update Layer0 instances.

Usage#

l0-setup apply [--force] [--access_key=awsAccessKeyID] [--secret_key=awsSecretAccessKeyID] [--region=awsRegion] [--dockercfg=filepath] [--vpc=vpcID] prefixName

Required parameters#

prefixName
The name of a Layer0Prefix in an existing AWS stack. To learn more about creating a stack, see the installation guide

Optional arguments#

--force
Avoid prompting for a missing dockercfg file.
--access_key=awsAccessKeyID
The Access Key ID of an IAM user associated with the AWS stack in which you are creating the Layer0 instance.
--secret_key=awsSecretAccessKeyID
The Secret Access Key ID of an IAM user associated with the AWS stack in which you are creating the Layer0 instance.
--region=awsRegion
The AWS region in which the Layer0 instance resides.
--dockercfg=filepath
The path to a valid Docker configuration file (likely found at ~/.docker/config.json for Docker versions 1.7 or greater, or ~/.dockercfg for versions less than 1.7).
--vpc=vpcID
The ID of a VPC. If blank, l0-setup will create a new VPC.

Backup#

The backup command is used to back up your Layer0 configuration files to an S3 bucket. This command is most often used when migrating between versions of Layer0. The backup command also runs automatically every time you execute the l0-setup apply command.

Usage#

l0-setup backup prefixName

Required parameters#

prefixName
The name of the Layer0Prefix that you want to back up.

Destroy#

The destroy command is used to delete a Layer0 configuration.

Before you can run the destroy command, you must first delete any existing loadbalancers, services and environments that exist in the Layer0 instance, using the delete subcommands for each of those components.

Caution

Destroying a Layer0 instance cannot be undone; if you created backups of your Layer0 configuration using the backup command, those backups will also be deleted when you run the destroy command. The only way to re-deploy a destroyed instance is to use the procedures in the installation guide to rebuild it.

Usage#

l0-setup destroy prefixName

Required parameters#

prefixName
The name of a Layer0Prefix in an existing AWS stack.

Optional arguments#

--force=false
When specified, destroy confirmation prompts will not be shown.

Endpoint#

The endpoint command is used to look up the details of a Layer0 endpoint so that you can export them to your shell.

Usage#

l0-setup endpoint [-diq] [-s syntax] prefixName

Required parameters#

prefixName
The name of a Layer0Prefix for which you want to view endpoint information.

Optional arguments#

-d, --dev=false
Show configuration variables required for local development
-i, --insecure=false
Allow incomplete SSL configuration. This option is not recommended for production use.
-q, --quiet=false
Silence CLI and API version mismatch warning messages
-s, --syntax="bash"
Show commands using the specified syntax (bash, powershell, cmd)

Plan#

The plan command is used to dry-run an apply, displaying what would happen without actually making any changes.

Usage#

l0-setup plan prefixName terraformArguments

Required parameters#

prefixName
The name of a Layer0Prefix for which you want to plan an update.

Optional arguments#

terraformArguments
The terraform arguments to pass to Layer0. Arguments passed in this way must follow the syntax specified in the Terraform Commands (CLI) Documentation.

Restore#

The restore command is used to restore Layer0 configuration files that were previously backed up to an S3 bucket using the backup command.

Usage#

l0-setup restore [--access_key=awsAccessKeyID] [--secret_key=awsSecretAccessKeyID] [--region=awsRegion] prefixName

Required parameters#

prefixName
The name of the Layer0Prefix that you want to restore.

Optional arguments#

--access_key=awsAccessKeyID
The Access Key ID of an IAM user associated with the AWS stack in which the Layer0 instance was created.
--secret_key=awsSecretAccessKeyID
The Secret Access Key ID of an IAM user associated with the AWS stack in which the Layer0 instance was created.
--region=awsRegion
The AWS region in which the Layer0 instance resides.

Terraform#

The terraform command is used to issue terraform commands directly to a Layer0 instance. Some terraform commands enable functionality that is not a standard part of the l0-setup application.

For more information about the capabilities and syntax of terraform commands, see the Terraform Commands (CLI) Documentation.

Usage#

l0-setup terraform prefixName terraformArguments

Required parameters#

prefixName
The name of a Layer0Prefix in which you want to execute terraform commands.
terraformArguments
The terraform arguments to pass to Layer0. Arguments passed in this way must follow the syntax specified in the Terraform Commands (CLI) Documentation.

VPC#

The vpc command is used to look up details from a Virtual Private Cloud (VPC) instance.

Usage#

l0-setup vpc [--access_key=awsAccessKeyID] [--secret_key=awsSecretAccessKeyID] [--region=awsRegion] prefixName

Required parameters#

prefixName
The name of a Layer0Prefix for which you want to view VPC information.

Optional arguments#

--access_key=awsAccessKeyID
The Access Key ID of an IAM user associated with the AWS stack in which you are creating the Layer0 instance.
--secret_key=awsSecretAccessKeyID
The Secret Access Key ID of an IAM user associated with the AWS stack in which you are creating the Layer0 instance.
--region=awsRegion
The AWS region in which the Layer0 instance resides.
